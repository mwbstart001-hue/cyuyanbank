# 银行管理系统核心漏洞专项测试报告

> 本文档收录了针对“数据校验缺失、违规重复销户、未结清销户、已销户跨行转账、风控拦截形同虚设”的 5 大专项测试，并汇总了预期与真实系统运行快照。

## 测试用例 1：数据基本校验全无（无效金额防呆防护）
- **测试操作流**: 输入 3(开户) -> 姓名 'TestA' -> 余额 -100
- **预期结果**: 系统拦截非法输入，提示"【错误】金额必须是非负数！开户中断。"
- **系统实际执行快照**:
```text
输入客户姓名: 输入初始余额 (>=0): 【错误】金额必须是非负数！开户中断。
```

> ✅ **测试结果结论: 与预期一致，漏洞修复有效！**
---

## 测试用例 2：一个账户能销两次（重复销户拦截）
- **测试操作流**: 输入 3(开户,余额0) -> 5(注销账户1) -> 5(再次尝试注销账户1)
- **预期结果**: 第一次注销成功。第二次注销被拦截，提示"【业务拦截】销户失败: 账户 ID 1 已经处于注销状态! 操作取消。"
- **系统实际执行快照**:
```text
输入客户姓名: 输入初始余额 (>=0): 输入初始信用分 (0-1000): ▶ 开户成功！分配的账户 ID 为: 1
输入要注销的账户 ID: ▶ 账户 ID [1] 注销成功！
输入要注销的账户 ID: 【业务拦截】销户失败: 账户 ID 1 已经处于注销状态! 操作取消。
```

> ✅ **测试结果结论: 与预期一致，漏洞修复有效！**
---

> ✅ **测试结果结论: 与预期一致，漏洞修复有效！**

## 测试用例 4：销了的账户还能转账（已注销账户交易阻断）
- **测试操作流**: 创造账户1和账户2(余额0)。注销账户2。尝试账户2向账户1转账，尝试账户1向账户2转账。
- **预期结果**: 两次转账全部失败。分别提示"转账失败: 转出账户已被注销，禁止转出!" 和 "转账失败: 接收账户已被注销，禁止转入!"
- **系统实际执行快照**:
```text
输入客户姓名: 输入初始余额 (>=0): 输入初始信用分 (0-1000): ▶ 开户成功！分配的账户 ID 为: 1
输入客户姓名: 输入初始余额 (>=0): 输入初始信用分 (0-1000): ▶ 开户成功！分配的账户 ID 为: 2
输入要注销的账户 ID: ▶ 账户 ID [2] 注销成功！
转出账户 ID: 转入账户 ID: 转账金额: 【转账失败】转账失败: 转出账户已被注销，禁止转出!
转出账户 ID: 转入账户 ID: 转账金额: 【转账失败】转账失败: 接收账户已被注销，禁止转入!
```

> ✅ **测试结果结论: 与预期一致，漏洞修复有效！**
---

## 测试用例 5：逻辑不够严谨的风控模型（动态风险拦截及惩罚体系）
- **测试操作流**: 创建信用极低的高风险账户(信用分500)。尝试发起8000元的大额风险转账。
- **预期结果**: 转账触发风控规则拦截：提示属于高风险客户、直接拒绝大于5000的转账，并执行【风控惩罚】扣除其 10 分信用分。
- **系统实际执行快照**:
```text
输入客户姓名: 输入初始余额 (>=0): 输入初始信用分 (0-1000): ▶ 开户成功！分配的账户 ID 为: 1
输入客户姓名: 输入初始余额 (>=0): 输入初始信用分 (0-1000): ▶ 开户成功！分配的账户 ID 为: 2
转出账户 ID: 转入账户 ID: 转账金额: ⚠ 【风控拦截】此账户 (2) 属于高风险客户！
⚠ 【风控拒绝】因信用风险，拒绝大于 5000 元的转账请求。当前请求：8000 元
⚠ 【风控惩罚】已扣除该账户 10 分信用分作为惩罚，当前信用分: 490
```

> ✅ **测试结果结论: 与预期一致，漏洞修复有效！**
