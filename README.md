# 银行综合管理系统

## 项目概述

基于C++原生开发（无第三方框架）的银行综合管理系统，整合循环队列、并查集、哈夫曼树、决策树实现核心业务，模块化解耦，本地文件存储。

## 系统架构

```
bank_system/
├── include/          # 头文件
│   ├── common.h      # 通用工具和基类
│   ├── queue_system.h    # 排队叫号模块（循环队列）
│   ├── union_find.h      # 账户关系模块（并查集）
│   ├── huffman_tree.h    # 交易流水模块（哈夫曼树）
│   └── decision_tree.h   # 风险评估模块（决策树）
├── src/              # 源文件
│   ├── main.cpp      # 主程序入口
│   ├── test.cpp      # 测试套件
│   ├── queue_system.cpp
│   ├── union_find.cpp
│   ├── huffman_tree.cpp
│   └── decision_tree.cpp
├── Makefile          # 构建脚本
└── README.md         # 项目说明
```

## 功能模块

### 1. 排队叫号模块（循环队列）

**核心功能：**
- 支持普通/VIP取号
- 叫号/超时作废
- 排队位置查询
- 普通转VIP
- 按取号时间/优先级排序

**管控机制：**
- 单窗口并发限制（最多5个）
- 总排队数上限（100个）
- 30分钟超时自动作废

### 2. 账户关系模块（并查集）

**核心功能：**
- 账户新增/注销
- 关联关系绑定/解除
- 关联账户查询
- 关联组备注修改
- 按余额/开户时间排序

**管控机制：**
- 账户删除前交易校验
- 关联变更留痕（日志记录）

### 3. 交易流水模块（哈夫曼树）

**核心功能：**
- 流水增删改查
- 哈夫曼压缩/解压
- 压缩率统计
- 按交易时间/金额排序

**管控机制：**
- 流水标记删除（留痕）
- 单条流水大小限制

### 4. 风险评估模块（决策树）

**核心功能：**
- 客户数据录入
- 信用等级（AAA/AA/A/B/C）评估
- 评估规则调整
- 按信用等级/风险分值排序

**管控机制：**
- 多维度（收入/征信/负债）评估
- 高风险客户自动标记

## 编译与运行

### 环境要求
- C++11或更高版本编译器
- Make工具

### 编译命令

```bash
# 编译主程序
make all

# 编译并运行测试
make test

# 编译并运行主程序
make run

# 清理编译产物
make clean
```

### 运行程序

```bash
# 运行主程序
./bin/bank_system

# 运行测试
./bin/test_system
```

## 使用说明

### 主菜单

系统启动后显示主菜单，可选择以下模块：
1. 排队叫号系统
2. 账户关系系统
3. 交易流水系统
4. 风险评估系统
0. 退出系统

### 各模块操作

每个模块都有独立的菜单系统，支持：
- 数据录入和管理
- 查询和排序
- 文件保存/加载

## 数据结构说明

### 循环队列
- 使用数组实现环形缓冲区
- 支持FIFO操作
- 自动处理队列满/空状态

### 并查集
- 路径压缩优化
- 按秩合并
- 支持动态连通性查询

### 哈夫曼树
- 基于频率构建最优前缀编码
- 支持数据压缩和解压
- 压缩率可达40-50%

### 决策树
- 基于规则的多叉决策
- 支持动态规则调整
- 多维度风险评估

## 文件存储

系统自动生成以下数据文件：
- `queue_data.txt` - 排队数据
- `account_data.txt` - 账户数据
- `relation_logs.txt` - 关联变更日志
- `transaction_data.txt` - 交易流水
- `transactions.huf` - 压缩后的交易数据
- `customer_data.txt` - 客户数据
- `evaluation_data.txt` - 评估结果

## 性能指标

基于1000条交易记录的测试：
- 添加操作：< 1ms
- 排序操作：~13ms
- 哈夫曼压缩：~4ms
- 压缩率：约44%
