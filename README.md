# 银行管理系统 (Bank Management System) - C++ & SQLite

这是一个基于 C++ 开发的银行管理原型系统，旨在通过真实的业务场景（排队、账户关联、风险评估、日志压缩）帮助零基础学习者掌握 C++ 核心算法、数据库操作及 Docker 部署。

---

## 1. 项目文件目录

```text
/
├── include/
│   ├── Account.h          # 账户实体类
│   ├── CircularQueue.h    # 循环队列（排队叫号逻辑）
│   ├── DatabaseManager.h  # SQLite 数据库集成与 CRUD
│   ├── DecisionTree.h     # 决策树（风险评估模型）
│   ├── Huffman.h           # 哈夫曼编码（日志压缩）
│   └── UnionFind.h        # 并查集（账户关联分析）
├── src/
│   └── main.cpp           # 系统入口与交互逻辑
├── data/
│   └── bank.db            # 自动生成的本地数据库文件
├── Makefile               # 自动化构建脚本
├── Dockerfile             # 镜像构建定义
└── docker-compose.yml     # 容器服务编排
```

## 3. 部署指南

### 第一步：进入项目目录
```bash
cd label-3248
```

### 第二步：Docker 部署指令

- **一键启动并进入系统 (支持键盘输入)**:
  若要直接构建并运行具有交互菜单的系统，请执行：
  ```bash
  docker compose run --build --rm bank_app
  ```

- **停止并清理**:
  ```bash
  docker compose down
  ```

- **重新启动 (不重构镜像)**:
  ```bash
  docker compose start
  ```

- **重启服务 (应用配置更改)**:
  ```bash
  docker compose restart
  ```

## 4. 最新漏洞修复与更新日志 (Update Log)

在最新的重构版本中，我们针对之前系统中存在的严重逻辑漏洞进行了全面修复，并引入了更严谨的 C++ 类型安全特性：

1. **引入 `AccountStatus` 强类型枚举**
   - **修复方案**：不再使用模糊的 `int` 表示状态，使用 `enum class AccountStatus` 管控账户生命周期（ACTIVE/CLOSED）。
2. **极致的销户前置校验**
   - **修复方案**：在 `DatabaseManager::deleteAccount` 中增加了余额结清校验。若账户余额不为 0，系统将硬性拦截销户操作，防止坏账遗留。
3. **实战化的决策树风控拦截**
   - **修复方案**：决策树不只是“说说而已”，它已深度嵌入 `transfer` 业务流。高风险客户若尝试大额转账（>5000元），系统不仅会拒绝交易，还会执行信用扣分惩罚（-10分）。
4. **数据库原子事务保护**
   - **修复方案**：重构并增强了 `transfer` 机制，在 SQLite 事务中实现转出/转入双向校验，并彻底阻断涉及“已注销”账户的任何资金行为。
5. **算法闭环与健壮性提升**
   - **修复方案**：补齐了哈夫曼（Huffman）解压流程；并查集（UnionFind）改用动态 Map 结构防止 ID 溢出；修复了所有潜在的类型转换编译警告。

## 5. 自动化集成漏洞测试验证指南

项目可以通过一行命令完成包含所有防护验证的自动化测试全流程：

### 自动化流测试指令
系统将自动执行：开户 -> 验证排序机制 -> 触发透支防线 -> 正常小额转账 -> 触发风险防线(高风险禁止大额转账) -> 正常销户 -> 触发重复销户防线 -> 触发死账户转账防线 -> 反洗钱追查(并查集) -> 日志无损压缩还原(哈夫曼)。

我们在项目根目录预置了 `test_input.txt` 文件，它包含了全套完整业务流程的键盘输入模拟。为了**兼容 Windows (CMD/PowerShell) 和 Mac/Linux 系统**，请直接使用文件重定向进行测试：

```bash
docker exec -i bank_management_system ./bank_system < test_input.txt
```

*(免责注：在 Windows 平台上，如果 `docker exec` 不直接支持输入重定向，您也可以直接在本地运行编译好的 `bank_system < test_input.txt`)*
